The output is a CSV instead of other storage choice such as a relational DB because:
- catalog is small/medium, only covering 14 categories from WOMAN new collection
- single user? (for now i am the only user)
- not rescrapping or updating often 
- relationalDB is more production-like system

cleaned product metadata is stored in Apache Parquet format to preserve data types, improve I/O efficiency, and support repeated filtering and ranking operations during recommendation.

CSV: Row-based, Must read the entire file to access any column, Slow for large files, Expensive when repeatedly loading for filtering and ranking
Parquet: Columnar format, You can load only the columns you need, Much faster for: filtering by category/price/color, repeated reads in recommender loops, Better for memory usage
